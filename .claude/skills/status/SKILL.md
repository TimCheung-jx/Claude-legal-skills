# /status

1. 加载配置 → 督导风格、简明语言标准、管辖地。
2. 使用下方工作流程。读取案件记录。
3. 为指定受众生成：
   - `client` — 简明语言，发生了什么/下一步/您做什么/联系我们
   - `internal` — 程序姿态、上次检查以来完成、 upcoming、需要指导律师输入、实习生评估
   - `court` — 按地方法院规则的正式状态报告格式
4. 按受众进行督导路由（客户面向和法院就绪通常标记）。

```
/status client
```


# 状态：受众感知的案件摘要

## 目的

团队生成大量状态更新 — 给客户、给指导律师、给合作律师、给法院。同一案件、同一事实、完全不同的文档。本技能获取案件记录并为正确的读者产生正确的摘要。

## 加载上下文

配置 → 督导风格、简明语言标准（客户面向）、管辖地。
案件记录获取事实。

## 受众模式

### 客户面向

**读者：** 客户。可能 stressed。可能不熟悉法律程序。阅读水平按配置中的简明语言标准（默认 6 年级）。

**包括：**
- 自上次他们听到团队消息以来发生了什么
- 下一步是什么以及何时
- 他们（如有）需要做什么
- 如何联系团队

**不包括：**
- 法律分析（他们不需要知道 IRAC）
- 他们案件的弱点（除非是时候进行那次对话 — 那是指导律师的决定，非状态更新）
- 术语

*给实习生的审阅标签（非客户 — 发送前删除）：*
`[AI辅助草稿 — 需要实习生审阅和按配置督导步骤]`

检查您管辖区的实习律师规则，了解实习生签署函件所需的披露语言；某些管辖区要求特定形式。

```markdown
尊敬的 [客户]：

想向您更新您的案件。

**发生了什么：** [ plain English。"我们于 [日期] 向法院提交您的答辩" 而非 "已提交答辩状。"]

**下一步：** [什么和何时。"法院安排了 [日期] [时间] 的听证。您需要到场。" 或："我们等待对方律师回应。那可能需要几周。"]

**您需要做什么：** [具体清晰。或："目前无需做任何事 — 我们需要时会通知您。"]

**如何联系我们：** [团队电话、工作时间、实习生姓名]

[实习生姓名]
实习律师
在 [指导律师] 的监督下
[团队名称]
```

**发送前：** 向客户发送状态更新是重要行为。门控是配置中的督导工作流程，由确认持牌督导律师拥有团队设置的角色检查加强。确认草稿已按督导协议（队列 / 标记 / 轻触式）审阅，且所有内部审阅标签（`[AI辅助草稿]`、`[核实]` 等）已从客户副本中删除。

### 内部（给指导律师）

**读者：** 督导指导律师。懂法律。想知道案件在哪里以及实习生需要什么。

**包括：**
- 程序状态（案件生命周期中的位置）
- 自上次检查以来完成的内容
- upcoming（期限、听证）
- 需要指导律师输入的问题
- 实习生的评估（进展如何、关切）

```markdown
# 状态：[客户] — [事项] — [日期]

**实习生：** [姓名] | **程序姿态：** [提交前 / 答辩已提交 /
发现 / 动议待决 / 等]

## 自上次检查以来

- [完成的内容]

## upcoming

| 日期 | 什么 | 需要行动 |
|---|---|---|
| [日期] | [期限/听证] | [日期] |

## 需要指导律师输入

- [问题或决策点 — 具体]

## 实习生的评估

[进展如何。优势、关切、策略问题。这是实习生的
思考展示的地方。]

---
[AI辅助草稿 — 实习生应特别修订评估部分；
那是您的思考，非记录摘要]
```

### 法院就绪

**读者：** 法官或书记员。正式。特定于法院需要什么（通常是法院命令的状态报告，或状态会议前的陈述）。

**包括：**
- 程序历史（简要）
- 发现/动议/和解的当前状态
- 什么 outstanding
- 建议的下一步或排期

**格式：** 按地方法院规则。标题、签署块、如提交的送达证书。

```markdown
═══════════════════════════════════════════════════════════════════════
  AI辅助草稿 — 需要实习生分析和律师审阅
  法院提交在提交前始终需要指导律师审阅
═══════════════════════════════════════════════════════════════════════

[按管辖地的标题 — 针对当前地方法院规则核实]

状态报告

[当事人] 谨此提交本状态报告，依据 [法院
[日期] 的命令 / 地方法院规则 [X] / 预定于 [日期] 的
状态会议前]。

1. 程序历史：[简要]

2. 当前状态：[发现状态 / 动议状态 / 和解状态]

3. outstanding 事项：[待决内容]

4. 建议的下一步：[排期，如法院需要输入]

[签署块 — [指导律师] 监督下的实习律师]

[如提交的送达证书]

---

[核实：标题格式、当地状态报告要求、送达
要求 — 按当前 [法院] 规则]
```

## 督导路由

按配置：
- 客户面向 → 通常是标记触发器（客户沟通）
- 内部 → 无标记（反正给指导律师）
- 法院就绪 → 如启用正式队列则始终标记（法院提交）

## 本技能不做什么

- **决定告诉客户什么。** 尤其在坏消息或案件弱点上 — 那是实习生和指导律师要先进行、然后实习生与客户进行的对话。状态更新是状态，非策略建议。
- **向法院提交任何东西。** 起草文档；指导律师审阅；按团队程序提交。
- **替代内部状态中的实习生评估。** "实习生的评估"部分是实习生的思考 — 草稿可以搭建它但不能写它。

## 以下一步决策树结束

根据本技能刚产出的内容定制选项结束。五个默认分支（起草 X、升级、获取更多事实、观望、其他）是起点，非锁定。树是输出；律师挑选。
