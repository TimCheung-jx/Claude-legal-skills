# 客户函件：常规函件

## 目的

律所发送大量常规函件："您的预约是周二下午2点"、"请带上您的租赁合同"、"我们已向法院提交您的答辩"。本技能从模板处理这些，让实习生不用每周都打同样的信。

**范围：仅限常规。** 实质性建议、坏消息、案件策略 — 那些是 `/status client` 或先与指导律师的对话，非模板信。

## 加载上下文

配置 → 简明语言标准、督导风格、律所联系信息。

## 教学法检查

读取业务领域指南，检查 `pedagogy_posture` 设置：

- **`guide` (默认)：** 产生结构和清单。让实习生起草每个部分。给反馈。
- **`assist`：** 产生函件。标记供实习生审阅。实习生通过审阅编辑学习。
- **`teach`：** 不产生函件。让实习生起草。给反馈。只在两次尝试后展示示范段落。

无论何种姿势，输出始终包括："**教学法模式：[assist/guide/teach]** — 由您的督导指南设置。"

## 签署和实习生-律师披露

检查您管辖区的实习律师规则，了解实习生签署函件所需的披露语言。模板使用通用形式 — 在发送前根据您的规则调整签署。

## 函件类型

> **审阅标签放在函件外面。** `[AI辅助草稿 — 需要按配置督导步骤审阅]` 标签是给实习生的提示，非函件正文部分。放在渲染模板上方，切勿放入客户看到的正文中。

### 预约确认

**日期：** [日期]

*给实习生的审阅标签（非客户 — 发送前删除）：*

```markdown

```

## 发送前

向客户发送函件是重要行为。在发送上述任何函件前，确认：

1. 草稿已按督导协议审阅。
2. 所有内部审阅标签（`[AI辅助草稿]`、任何 `[核实]` 或 `[需要事实]` 标签）已从客户副本中删除。
3. 签署符合您管辖区的实习律师规则。

**这是给督导律师审阅的实习生草稿，非最终函件。** 发送它对客户有法律后果，可能构成法律建议或代表客户的沟通。持牌督导律师在函件离开律所前审阅、编辑并批准。未经督导批准不得发送。

## 简明语言检查

短句。无术语。阅读水平目标执行。如模板包含客户可能不知道的法律术语，首次解释："我们提交了您的'答辩' — 那是告诉法院您这边情况的文件。"

## 督导路由

常规函件可能是也可能不是标记触发器，取决于指导律师选择的督导风格。如轻触式：这些在实习生审阅后直接发出，无需队列步骤。如正式队列：即使常规函件也排队。

## 本技能不做什么

- **实质性建议。** 如函件会说"我认为您的案子如何"或"您应该怎么做"，那不是常规 — 那是 `/status client` 或先与指导律师的对话。
- **坏消息。** 案件结案、不利裁决、无法帮助 — 那些需要思考，非模板。标记给指导律师。
- **给对方的任何函件或给法院的文件。** 不同受众，不同技能 (`/draft` 或 `/status court`)。
