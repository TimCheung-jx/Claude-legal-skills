# 客户接待

## 目的

接待是律所/法务团队最大的瓶颈之一。实习生可能花45分钟访谈，再花一小时撰写，更多时间发现问题。 meanwhile 等待名单在增长。

本技能结构化对话、生成撰写稿、发现问题、标记冲突 — 让实习生的时间用于分析，而非转录。

**不做什么：** 决定是否接受案件。那是实习生的分析和指导律师的判断。Claude 加速信息收集和结构化，而非 lawyering。

## 加载上下文

配置 → 业务领域、接待模板（如多个）、督导风格、管辖地、标记触发器。

## 读取指导律师指南

检查是否有业务领域指南。如存在，使用其接待问题、红旗标记、适配标准；如不存在，使用通用默认并在接待摘要末尾注明。

## 工作流程

### Step 1: 业务领域路由

该接待从哪个业务领域开始？客户可能不知道 — 他们知道问题，不知道法律类别。

> "请告诉我发生了什么 — 什么让您来到我们这里？"

从回答中路由到适当的接待模板。如律所处理多个领域且问题跨越（房产客户提及劳动纠纷，家事客户提及债务问题），记录所有相关领域。

### Step 2: 业务领域特定接待

每个领域问不同问题。使用配置中的模板。默认模板如下：

**房产/租赁：**
- 房产类型（商品房、公租房、城中村）
- 发生了什么：收到通知、被换锁、条件问题、押金纠纷
- 租赁合同条款和付款记录
- 居住条件问题（维修请求、房东回应、证据）
- 时间紧迫性：通知日期、法院日期

**家事：**
- 关系和争议焦点（抚养权、赡养费、离婚、保护）
- 涉及子女 — 年龄、当前安排
- 安全：任何暴力、威胁、恐惧（谨慎处理）
- 现有法院命令
- 时间紧迫性：任何已安排的听证

**劳动：**
- 雇佣关系类型（合同工、正式工、劳务派遣）
- 发生了什么：违法解除、欠薪、工伤、社保纠纷
- 劳动合同条款和工作年限
- 已采取的步骤：劳动仲裁申请、调解
- 时间紧迫性：仲裁时效、举证期限

**债务/执行：**
- 债务类型或争议
- 谁在联系他们以及如何（催收 relevance）
- 文件：合同、对账单、催收函
- 是否有任何已提交的诉讼
- 时间紧迫性：答辩期限、执行期限

### Step 3: 跨业务领域问题发现

在运行领域模板时，监听领域外的问题：

| 客户提到 | 同时标记 |
|---|---|
| "我担心我的劳动权益" | 劳动法问题 — 即使在房产接待中 |
| "我的伴侣[威胁行为]" | 家暴/家事法/保护令 — 即使在债务接待中 |
| "我因为受伤不能工作" | 可能的工伤/人身伤害索赔 |
| "他们从我的工资里扣钱" | 工资扣除 — 劳动/执行重叠 |
| "房东说他会让我搬走" | 房产 + 可能的报复索赔 |

在摘要中记录每个跨领域问题。律所可能处理、转介或两者 — 那是指导律师的决定。实习生应该看到它。

### Step 4: 利益冲突检查标记

根据配置中的利益冲突检查流程。至少：

- 对方当事人姓名 — 律所是否代表或曾代表过他们？
- 相关方 — 实习生或律所可能与之有冲突的其他人？
- 立场冲突 — 此案是否要求做会损害另一客户的事？

标记给指导律师审阅。不解决冲突 — 只是提出。

### Step 5: 分流分类

非案件接受决定 — 分流输入：

| 分类 | 含义 |
|---|---|
| **紧急** | 几天内有截止日期、安全问题、不可逆损害迫在眉睫 |
| **时间敏感** | 几周内有截止日期、损害持续但非立即不可逆 |
| **标准** | 无立即截止日期，可正常排队 |
| **可能超出范围** | 问题超出律所业务领域 — 标记转介评估 |

### Step 6: 督导标记检查

根据配置中的督导风格和标记触发器。如正式队列或可配置标记启用，且存在触发器（提及截止日期、家暴指示、劳动权益问题等），记录标记。

### Step 7: 期限交接 — 必需交付物

如接待发现任何时间线截止日期（答辩期限、听证、诉讼时效截止、整改期、申请窗口、通知窗口），**在接待输出中发出可直接复制粘贴的 `/deadlines --add ...` 代码块**。这是必需交付物，非建议。

格式：

```
/deadlines --add
  case=[案件简称]
  type=[response|hearing|statute-of-limitations|discovery|cure-period|filing-window|notice|other]
  description="[截止日期内容的一行描述]"
  due=[核实 — 实习生+指导律师从触发事件计算]
  source="[触发事件 + 法规/规则引用]"
  owner=[实习生姓名]
  warnings=[14,7,3,1]
```

规则：
- 每个截止日期一个块。不要合并。
- 如截止日期是管辖性的（答辩期限、诉讼时效、通知窗口），将 `due=` 留为 `[核实]`。
- 如日期在触发文件中给出（传票上的听证日期），填入该日期。
- 如未发现截止日期，省略此部分。

## 本技能不做什么

- **决定案件接受。** 实习生分析，指导律师决定。
- **解决冲突。** 标记给指导律师。
- **在接待中提供建议。** 接待是收集；建议来自分析和指导律师审阅后。
- **生成最终文件。** 摘要是起点 — 实习生阅读、纠正任何误述，并从中建立分析。
